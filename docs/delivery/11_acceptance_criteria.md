# 11 — Acceptance criteria

## MVP acceptance criteria

The prototype is acceptable only within a declared acceptance profile. See `docs/delivery/16_acceptance_profiles.md`.

The checklist below is the default engineering prototype target, not a blanket proof of Suno parity.

### Model and pipeline

- [ ] At least one real local inference backend is integrated for real-local-inference acceptance.
- [ ] A mock backend exists for API and CI tests.
- [ ] Model/backend choice is documented.
- [ ] External music-generation SaaS is not used as the generation backend.
- [ ] Russian lyrics are generated or accepted.
- [ ] Audio artifact is generated by the selected local backend, unless the declared profile is API/mock only.
- [ ] The actual output mode is recorded and does not overclaim lyric/vocal alignment.
- [ ] Metadata is saved for each job.

### API

- [ ] `POST /v1/tracks` creates a job.
- [ ] `GET /v1/tracks/{job_id}` returns status.
- [ ] `GET /v1/tracks/{job_id}/lyrics` returns lyrics.
- [ ] `GET /v1/tracks/{job_id}/audio` returns audio.
- [ ] `GET /v1/models` returns available model information.
- [ ] `GET /health` returns service status.
- [ ] OpenAPI/Swagger documentation exists.

### Reproducibility

- [ ] README explains how to run locally.
- [ ] Dependency setup is documented.
- [ ] Dockerfile or equivalent reproducible setup exists.
- [ ] CPU/mock fallback exists for API tests if GPU is unavailable.

### Benchmark

- [ ] Fixed prompt set exists.
- [ ] Scoring rubric exists.
- [ ] Reference baseline method is documented.
- [ ] Local outputs are scored.
- [ ] Verdict is explicit and does not overclaim.
- [ ] Benchmark scope states prompt set, evaluator mode, reference date, and unsupported claims.

### Error handling

- [ ] Invalid input returns structured error.
- [ ] Model runtime error returns structured error.
- [ ] Missing artifact returns structured error.
- [ ] Worker failure is visible in job status.

## Bonus acceptance criteria: GPU execution / GPU farm

- [ ] API can dispatch to a worker through a queue or scheduler.
- [ ] Worker runs on GPU or exposes a testable GPU execution abstraction.
- [ ] Worker metadata includes scheduler/worker/device fields.
- [ ] Artifacts survive worker termination.
- [ ] Cold-start or warm-pool behavior is documented.

## Explicit non-acceptance cases

The assignment is not acceptable if:

- It merely wraps Suno or another external generator.
- It claims Suno parity without a benchmark report.
- It presents a plan as if it were a working system.
- It says fine-tuning was done without training logs and before/after evaluation.
- It ignores model license constraints.
- It cannot be run or tested in any form.
- It counts mock output as proof of real local AI generation.
- It claims sung Russian lyrics when only a lyrics file or conditioning prompt exists.
